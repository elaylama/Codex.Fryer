# CODEXFRYER

`CODEXFRYER` — публичный репозиторий со скиллом формата **grill-me**, адаптированным специально под **Codex**.

Цель скилла: до начала реализации провести жёсткий структурированный допрос идеи, снять неоднозначности и превратить «смутный замысел» в чёткий, исполнимый план.

## Почему это именно скилл для Codex

- Скилл учитывает, что Codex умеет работать прямо в кодовой базе и может сам добывать ответы из репозитория.
- Логика допроса завязана на инструментальный режим Codex: сначала факты из проекта, потом вопросы пользователю.
- Результат скилла ориентирован на следующий шаг в Codex: конкретный старт имплементации, а не абстрактные рассуждения.

## Что делает скилл

1. Уточняет контекст задачи и ограничения.
2. Идёт по веткам решения по одной, не перепрыгивая.
3. Задаёт вопросы по одному, с объяснением «зачем этот вопрос».
4. Даёт рекомендуемый вариант ответа для ускорения выбора.
5. Каждые 8-10 вопросов подводит промежуточный итог.
6. В конце формирует мини-спецификацию и первый обратимый шаг реализации.

## Когда запускать

- Перед стартом нового проекта или фичи.
- Когда «вроде всё понятно», но есть риск упустить архитектурные детали.
- Когда в команде разные ожидания и нужен единый контракт до кодинга.
- Когда нужно быстро снизить риск переработок.

## Когда не запускать

- Если вы уже утвердили спецификацию и критерии успеха.
- Если задача микроскопическая и полностью детерминирована (например, механический rename в одном файле).

## Установка в Codex

### Вариант 1. Через script `skill-installer`

```bash
python3 ~/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py \
  --repo <ваш-github-логин>/CODEXFRYER \
  --path .
```

### Вариант 2. Через URL

```bash
python3 ~/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py \
  --url https://github.com/<ваш-github-логин>/CODEXFRYER/tree/main
```

После установки перезапустите Codex, чтобы скилл появился в активном списке.

## Как использовать в чате

Примеры триггеров:

- `[$codexfryer](./SKILL.md) Прогриль мою идею: B2B SaaS для автоматизации отчётов.`
- `Используй codexfryer и пройди со мной архитектуру Telegram-бота для саппорта.`
- `Давай grill-me режим перед тем как писать код.`

## Пример короткой сессии

**Пользователь:** «Хочу сервис, который автоматом проверяет SEO сайта».

**CODEXFRYER (Q1):** «Кто основной пользователь: SEO-специалист агентства, инхаус-маркетолог или владелец малого бизнеса?  
Рекомендованный ответ: начните с инхаус-маркетолога, чтобы быстрее сузить UX и отчётность».

**Пользователь:** «Инхаус-маркетолог».

**CODEXFRYER (Q2):** «Какой главный outcome: рост органики, снижение техдолга SEO или скорость аудита?  
Рекомендованный ответ: скорость аудита + список фиксируемых проблем с приоритетами».

… (допрос продолжается по одной ветке за раз).

Финал:

- цель
- ограничения
- не-цели
- критерии успеха
- первый минимальный обратимый шаг

## Структура репозитория

- `SKILL.md` — основной скилл для Codex.
- `examples/` — сценарии и примеры диалогов.
- `PUBLISHING.md` — чеклист публикации на GitHub.

## Практические советы

- Отвечайте конкретно, чтобы снизить количество итераций.
- Если у вас есть репозиторий, запускайте скилл в корне проекта.
- Не проскакивайте «не-цели» — именно они спасают от расползания скоупа.
- После финального резюме сразу просите Codex реализовать «первый обратимый шаг».
